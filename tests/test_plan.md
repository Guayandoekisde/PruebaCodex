# Checklist breve de pruebas

## 1) Checklist de pruebas mínimas
- [ ] El programa corre en un caso nominal.
- [ ] Se validó al menos una entrada inválida.
- [ ] Se documentó el resultado esperado vs obtenido.

## 2) Checklist de documentación
- [ ] `docs/propuesta.md` está completo.
- [ ] `docs/caso_de_uso.md` está completo.
- [ ] `docs/plan_de_pruebas.md` tiene al menos 3 casos.
- [ ] El alcance mínimo y fuera de alcance están claros.

## 3) Checklist de ejecución
- [ ] `scripts/run.sh` funciona en entorno local.
- [ ] Existe `src/main.<ext>` acorde al lenguaje elegido.
- [ ] No se usan dependencias complejas ni servicios externos.

## 4) Checklist de entrega final
- [ ] Estructura de carpetas correcta.
- [ ] Proyecto pequeño y viable.
- [ ] Todo está en español claro y técnico.
- [ ] Repositorio listo para revisión en GitHub Classroom.
